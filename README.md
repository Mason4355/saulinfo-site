# SaulInfo Site

Отдельный пользовательский сайт для `www.saulinfo.ru`.

Цели проекта:

- отдельный репозиторий и контейнер
- отдельный публичный домен
- интеграция с `shop-update` через аккуратный слой доступа к данным
- без вмешательства в текущую внутреннюю админскую систему

## Что внутри

- Flask-приложение для публичного сайта
- gateway-слой для чтения пользовательских данных из БД `shop-update`
- каркас VK-входа
- страницы кабинета пользователя
- GitHub Actions для автодеплоя на сервер

## Локальный запуск

```bash
cp .env.example .env
docker compose up --build
```

Открыть:

```text
http://localhost:8080
```

## Интеграция с shop-update

Сайт читает данные из SQLite-базы текущей системы через `SHOP_UPDATE_DB_PATH`.

По умолчанию в контейнер монтируется:

```text
/root/shop-update/data -> /integrations/shop-update
```

Это позволяет новому сайту работать поверх тех же пользователей, ключей, баланса и тикетов, не изменяя код `shop-update`.

## Деплой

После настройки GitHub Secrets деплой делается автоматически при push в `main`.
